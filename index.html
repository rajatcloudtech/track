6-Month Roadmap
Your journey from intermediate → high-value techno-functional consultant.
PHASES
Phase 1 · Months 1–2
Data Mastery
Deep-dive into SAP tables. Trace every PO end-to-end. Write SQL that detects real issues in EKBE, MSEG, MKPF. Goal: no developer dependency for data investigation.
Phase 2 · Months 3–4
Techno-Functional Shift
Learn to read ABAP. Set breakpoints. Understand BADIs, user exits, Z-tables. Build one Z-report from a real use case. Goal: solve 2 real issues without calling a developer.
Phase 3 · Months 5–6
Visible Expertise
Own one problem area (GR/IR, pricing, STO). Write one knowledge doc per month. Mentor a junior. Become the named go-to person on your project.
WEEKLY TARGETS
◷
Daily (45 min minimum)
15 min: one SAP concept or table · 15 min: write one SQL query · 15 min: document what you learned
◈
Weekly (1 real problem)
Pick one issue from your live project. Trace it fully. Write a 3-line summary: issue, root cause, fix.
▤
Monthly (1 knowledge doc)
Document a scenario you mastered. Include: symptom → tables checked → SQL used → resolution. This becomes your portfolio.
↗
The brutal reality
6 months of this → strong consultant. 1 year → high-value. Ignore it → 2–3 years of slow growth and frustration.

Debug Checklist
1. Confirm issue type (data / config / process)
2. Check header table first (EKKO)
3. Check item table (EKPO)
4. Check history (EKBE)
5. Check movement (MSEG with MBLNR)
6. Check FI doc (BKPF/BSEG if accounting)
